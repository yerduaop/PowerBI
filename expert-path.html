✏️ Exercice Pratique

🎯 Exercice 1.1 : Audit et Optimisation

Prenez un rapport existant et optimisez-le :

  1. Utiliser Performance Analyzer pour identifier visuels lents
  2. Analyser modèle avec VertiPaq Analyzer (DAX Studio)
  3. Supprimer colonnes inutilisées (réduction taille)
  4. Remplacer colonnes calculées par mesures
  5. Optimiser DAX avec variables
  6. Mesurer amélioration (temps chargement avant/après)
🎯 Objectif : Réduire taille modèle de 30% et temps de chargement de 50%

✏️ Exercice Pratique

🎯 Exercice 2.1 : RLS Multi-Niveaux

Implémentez une sécurité hiérarchique :

  1. Créer table UserAccess (Email, Level, Region, StoreID)
  2. Implémenter rôle dynamique avec SWITCH sur Level
  3. CEO voit tout, Regional voit région, Store voit magasin
  4. Tester avec "View as" pour chaque niveau
  5. Publier et assigner utilisateurs dans Service
  6. Créer mesures debug (Current User, My Regions)

✏️ Exercice Pratique

🎯 Exercice 3.1 : Fonction M Personnalisée

Créez une fonction pour automatiser import de fichiers :

  1. Fonction qui accepte chemin dossier en paramètre
  2. Liste tous fichiers CSV du dossier
  3. Combine tous fichiers en une seule table
  4. Ajoute colonne "SourceFile" avec nom fichier
  5. Gère erreurs (fichiers corrompus)
  6. Teste avec paramètres DEV/PROD

Projet Final : Solution Entreprise (4h)

🏆 Projet : Solution Power BI Complète

Créez une solution entreprise de A à Z :

🏗️ Architecture

  • Star Schema optimisé (< 500 MB)
  • 3 environnements (DEV/TEST/PROD) avec Deployment Pipeline
  • Paramètres gérés par environnement
  • RLS multi-niveaux implémenté

📊 Rapports

  • Dashboard Executive (5 KPI + tendances)
  • Analyse détaillée (visuels avancés, drill-through)
  • Page admin (usage metrics, performance)
  • Mobile layout pour toutes pages

🔧 Technique

  • Version control avec Git
  • Power Query M avec fonctions personnalisées
  • Incremental refresh configuré
  • Performance < 3 sec par page

📋 Documentation

  • Data Dictionary complet
  • Architecture diagram
  • User Guide PDF
  • Process de support défini
💡 Checklist Validation :
  • ✅ Tests UAT complétés avec users
  • ✅ RLS testé pour tous rôles
  • ✅ Performance validée (< 3 sec)
  • ✅ Documentation review par pairs
  • ✅ Backup et rollback plan
  • ✅ Formation utilisateurs effectuée
  • ✅ Monitoring configuré
